# Slipway acceptance evidence matrix

This matrix maps 35 product behaviors to reproducible artifacts and honest evidence gaps. It is an engineering evidence index, not a release verdict, Issue status, Run-routing input, Review result, or CLI progression input.

The executable Shell and native Windows assets are the repository's black-box end-to-end tests against a built Slipway binary. Fixtures, validators, prompt scenarios, and transcript instructions support the broader acceptance evidence and are not all E2E tests.

Evidence labels have exact meanings:

- **C** — deterministic Go contract/property/race test or static generated-template assertion; proves only CLI/core observable semantics or template text.
- **S** — Shell acceptance under this directory that invokes a built Slipway binary.
- **G** — isolated live GitHub.com fixture. Ownership-sensitive coverage records whether the repository is personal-account-owned or organization-owned. The local publication fault harness is adjacent evidence, not G.
- **H** — a real sanitized host prompt transcript plus evaluator notes.
- **W** — native Windows execution through both `cmd.exe` and PowerShell.
- **R** — documentation, website, package, or release validation.

C cannot substitute for H. A fake endpoint or this deterministic harness cannot substitute for live G. A Windows cross-build or static script scan cannot substitute for native W. `available` means the linked executable/static artifact exists in this tree; `not collected` means mapped runtime evidence has not been recorded; `external` means collection needs a protected account, host, or native platform outside this local run. Missing evidence is uncertainty and never controls CLI progression, merge, or release.

## R validation package

R does not replace the C/S/G/H/W evidence mapped to another execution boundary. Repository-side R artifacts include the three-locale [documentation](../../docs/en/index.md), language-neutral [machine schema](../../docs/reference/v2/machine-protocol.schema.json), [website navigation](../../website/astro.config.mjs), stdlib [link checker](link_check.py), reproducible [release artifact checker](release_artifacts.py), and CI/docs/release workflow collectors. The checker assets and workflow wiring are **available**. A successful invocation establishes only that invocation's R result; it is not a delivery certification.

## Commands

From the repository root, with a temporary binary outside the tree:

```bash
acceptance_tmp="$(mktemp -d)"
trap 'rm -rf "$acceptance_tmp"' EXIT
go build -o "$acceptance_tmp/slipway-acceptance-bin" .
SLIPWAY_BIN="$acceptance_tmp/slipway-acceptance-bin" sh tests/acceptance/machine-protocol.sh
SLIPWAY_BIN="$acceptance_tmp/slipway-acceptance-bin" sh tests/acceptance/adapters.sh
python3 -I tests/acceptance/github_publication_fault_harness.py
python3 -I tests/acceptance/verify_optional_release_checksums_test.py
python3 -I tests/acceptance/link_check.py --self-test
# After `npm --prefix website ci && npm --prefix website run build`:
python3 -I tests/acceptance/link_check.py --require-site
# After a GoReleaser snapshot populated dist/:
python3 -I tests/acceptance/release_artifacts.py --dist dist
```

See [Windows assets](windows/README.md), [live GitHub instructions](live-github/README.md), and the [sanitized transcript format](transcripts/README.md) for evidence that cannot be collected on an ordinary macOS/Linux run.
