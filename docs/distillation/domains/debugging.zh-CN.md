# Domain：debugging

| Skill | Tier | 主要绑定 |
|-------|------|----------|
| `root-cause-tracing` | T1 | host `wave-orchestration`；command `repair`；`technique-hint` |

作用：

1. 修复前必须有可复核的 root cause 说明。
2. 当初始追踪结论冲突时，进入 competing-hypothesis 分支。

说明：

- 吸收 `systematic-debugging`、`debugging-strategies`、`debug-buttercup` triage 姿态与 `parallel-debugging`。
- 绑定面较窄但保持 T1，因为它表达的是可复用方法而不是工具路由。
